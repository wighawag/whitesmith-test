## Description

Add a firework ASCII art animation to the existing CLI application. When the program is run, it should display an animated firework launching upward in the terminal, exploding into a burst pattern, and then revealing a randomly chosen ASCII art animal. The existing greeting behavior (`hello world` or `hello {name}`) must be preserved and displayed after the animation completes.

This feature was also requested in issue #17 but was never implemented. The current codebase is a simple TypeScript CLI (`src/index.ts`) that prints a greeting with a smiley emoji. The animation should be added to this file, wrapping existing logic in an async main function.

### Current state of `src/index.ts`
```typescript
#!/usr/bin/env node
const name = process.argv[2];
if (name) {
  console.log(`hello ${name} 😊`);
} else {
  console.log("hello world 😊");
}
```

## Acceptance Criteria

- Running `node dist/index.js` displays a firework animation in the terminal that visually "launches" upward through several frames.
- After the launch, an "explosion" burst pattern is displayed using ASCII characters (e.g., `*`, `·`, `✦`, `✧`).
- After the explosion, a randomly selected ASCII art animal is displayed. There must be at least 3 different animals in the pool (e.g., cat, dog, rabbit).
- A different animal may appear on each run (random selection from the pool).
- The existing greeting (`hello world 😊` or `hello {name} 😊`) is still printed after the animation completes.
- Running `pnpm build` succeeds with no TypeScript errors.
- The compiled `dist/index.js` file is included and up to date.
- No new npm dependencies are added — only Node.js built-in APIs are used.
- The `#!/usr/bin/env node` shebang remains at the top of the source file.

## Implementation Notes

### Files to modify
- **`src/index.ts`** — Add the firework animation logic and animal ASCII art, refactor to async main function.
- **`dist/index.js`** — Must be regenerated by running `pnpm build` after editing the source.

### Approach

1. **Define ASCII art animals** — Create an array of at least 3 multi-line string ASCII art animals. Suggested animals:

   Cat:
   ```
    /\_/\
   ( o.o )
    > ^ <
   ```

   Dog:
   ```
    / \__
   (    @\____
    /         O
   /   (_____/
   /_____/   U
   ```

   Rabbit:
   ```
    (\(\
    ( -.-)
    o_(")(")
   ```

2. **Create a `sleep(ms)` helper** using `new Promise<void>(resolve => setTimeout(resolve, ms))`.

3. **Create an async `playFirework()` function** that:
   - Uses `console.clear()` between frames to create animation effect.
   - Shows several frames (~5-8) of a firework launching upward. For example, a `|` character moving from the bottom of a text area toward the top, with a trail or spark character.
   - Shows 1-2 explosion frames with a starburst pattern made of ASCII characters spreading out from the top.
   - Shows the final frame with the randomly selected ASCII art animal centered in the explosion area.
   - Uses `sleep()` delays between frames (100-200ms per frame) so the total animation is ~2-3 seconds.

4. **Refactor `src/index.ts`** to use an `async function main()` pattern:
   ```typescript
   #!/usr/bin/env node

   const sleep = (ms: number) => new Promise<void>(resolve => setTimeout(resolve, ms));

   const animals: string[] = [
     // ... multi-line ASCII art strings
   ];

   async function playFirework(): Promise<void> {
     // animation frames with console.clear() and sleep()
     // pick random animal: animals[Math.floor(Math.random() * animals.length)]
   }

   async function main() {
     await playFirework();

     const name = process.argv[2];
     if (name) {
       console.log(`hello ${name} 😊`);
     } else {
       console.log("hello world 😊");
     }
   }

   main();
   ```

5. **Build** by running `pnpm build` to compile TypeScript and update `dist/index.js`.

### Conventions to follow
- The project uses TypeScript with `strict: true`, targeting `ES2020` with `Node16` module resolution (see `tsconfig.json`).
- The output goes to `dist/` directory (configured in `tsconfig.json` `outDir`).
- Keep the `#!/usr/bin/env node` shebang as the first line.
- No external dependencies — use only `console.clear()`, `console.log()`, `process.stdout.write()`, `setTimeout`, and `Math.random()`.
